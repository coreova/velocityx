# VelocityX Code Of Conduct

VelocityX is a Coreova project. Its public project spaces should be calm, respectful, useful, and safe for serious collaboration.

## Our Standard

Welcome behavior includes:

- Respecting different viewpoints, backgrounds, and levels of experience.
- Giving constructive feedback focused on the work.
- Asking clear questions and assuming good intent when context is incomplete.
- Helping keep issues, pull requests, and discussions scoped and useful.
- Respecting project boundaries, licenses, and maintainer time.

Unacceptable behavior includes:

- Harassment, threats, intimidation, or discriminatory language.
- Personal attacks, insults, repeated provocation, or hostile commentary.
- Publishing private information without explicit permission.
- Sexualized language or imagery in project spaces.
- Spam, misleading claims, or attempts to manipulate project processes.
- Any conduct maintainers reasonably consider unsafe or harmful.

## Scope

This code applies in VelocityX project spaces, including issues, pull requests, discussions, documentation, and public interactions where someone is representing the project.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, limit participation, or block users when behavior violates this code or disrupts the project.

Report conduct concerns through the least public appropriate GitHub channel available. Reports will be handled with care and reporter privacy will be protected where possible.
