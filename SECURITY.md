# Security Policy

Coreova takes VelocityX security reports seriously. VelocityX is published by Coreova, created by Maruf Ahmed Limon, and MIT licensed by Coreova.

## Reporting A Vulnerability

Do not report security vulnerabilities through public GitHub issues, pull requests, or discussions.

Use GitHub private vulnerability reporting when it is enabled:

1. Open `https://github.com/coreova/velocityx`.
2. Go to the `Security` tab.
3. Choose `Report a vulnerability`.
4. Include clear reproduction steps, impact, affected versions, and any safe proof of concept.

If private vulnerability reporting is not enabled, open a public issue only to ask for a private reporting path. Do not include vulnerability details in that issue.

## What To Include

- Affected VelocityX version or commit.
- Browser, operating system, and affected page or media player.
- Steps to reproduce.
- Expected impact and who could be affected.
- Known workarounds or mitigations.
- Whether the issue has been disclosed elsewhere.
- Whether you want public credit after resolution.

## Supported Versions

Until a separate support policy exists, assume only the latest public release and the default branch are actively reviewed.

## Handling

Maintainers should investigate valid reports privately, avoid unnecessary disclosure before a fix or mitigation is available, and credit reporters when appropriate and requested.
